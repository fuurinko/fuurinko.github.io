数据集介绍
glnn使用的数据集是cora.npz

  1. adj_data, adj_indices, adj_indptr, adj_shape:
    These arrays represent the adjacency matrix of the graph in the Compressed Sparse Row (CSR) format. In CSR format:

    • adj_data: Contains the non-zero values of the matrix.
    • adj_indices: Contains the column indices for each non-zero value.
    • adj_indptr: Contains the index in adj_data where each row starts.
    • adj_shape: Specifies the shape of the adjacency matrix.

    Together, these arrays allow you to reconstruct the adjacency matrix of the graph efficiently, especially when the graph is sparse.

  2. attr_data, attr_indices, attr_indptr, attr_shape:
    These arrays represent the node attribute matrix in the CSR format, similar to the adjacency matrix. The node attribute matrix specifies features or attributes for each node in the graph.

    • attr_data: Contains the non-zero values of the node attribute matrix.
    • attr_indices: Contains the column indices for each non-zero value in the attribute matrix.
    • attr_indptr: Contains the index in attr_data where each row of the attribute matrix starts.
    • attr_shape: Specifies the shape of the node attribute matrix.

  3. labels:
    This array contains the labels or classes for each node in the graph. It’s typically used in node classification tasks.

  4. node_names:
    This array contains the names or identifiers for each node in the graph. It helps in mapping the nodes to their corresponding attributes, labels, and positions in the adjacency matrix.

  5. class_names:
    This array contains the names or identifiers for each class or label in the dataset. It helps in understanding the meaning or context of each label in the labels array.

In summary, these arrays provide a comprehensive representation of a graph, including its structure (adjacency matrix), node attributes, node labels, and other metadata.

import

没有什么特别值得注意的地方

main

先大致梳理一遍流程，具体的函数会在后文详细分析
首先获取所有命令行参数。
根据num_exp来判断run函数需要执行多少次。
一次就调用一遍run，多次就调用repeat_run，并分别返回score或score与其标准差。
然后将score从数组的float类型转换为string类型并打印输出以及保存在exp_results文件中。
然后我们根据main的调用顺序依次分析被调函数。

get args()

基本上整个函数都是用argparse模块来从命令行获取函数运行时的参数。简单来说就是自定义一些命令行的可选参数使得脚本执行的时候更直观方便地直接在命令行中控制参数。
这个函数定义了很多参数，但是example中实际执行时需要规定的并不多，基本都是默认值，比较重要的需要用户自定义的例如 exp setting
内容没有必要细看，需要的时候对照查找就好。

run()

也是最关键的函数。

return

首先注释说明了返回值是测试集的评估分数。
且

set_seed

函数参数通过args.seed获取。
主要功能就是获取随机种子，其中会根据device是cpu或gpu进行一些调整。

获取device

接着检查gpu状态，如果有则设置device为cuda。

创建文件夹

根据指定的模式创建对应路径的输出文件所在的文件夹。
如果选择的是tran，则在指定路径下创建相对应的指定名称的文件夹，ind同理，但是文件夹名称会多出split_rate参数，具体功能在后面说明。

检测输出文件夹状态

接着大致检查一下输出文件夹是否可写，如果没什么问题就在终端输出文件夹对应路径。

load data

首先是调用了dataloader.py定义的load_data函数。
简单来说就是根据dataset的不同，又进入不同的特定的数据加载函数中，例如cpf数据集调用了load_cpf_data
读取数据部分，调用的是load_npz_to_sparse_graph(data_path),将返回值赋给data。
load_npz_to_sparse_graph(data_path)大概将就是读取npz的一些arrays，然后把arrays储存到各种对应的变量中。

接下来就是对读入的raw data的一些简单的处理。

standardize()

removing self-loops and extracting the largest connected component using the standardize method.

unpack()

默认的方法。

  • The adjacency matrix (adj), node features (features), and labels (labels) are then unpacked from the data.
    binarize_labels(labels)
    converts multi-class labels into a one-hot encoded format.

然后将刚刚处理的一些数据从sparse matrix format 转为 a dense numpy array继而转为tensor，包括g, labels, idx_train, idx_val, idx_test。
其中g = dgl.graph((adj_sp.row, adj_sp.col))是利用dgl包根据领接矩阵生成的图。
数据加载完成之后，打印加载的情况，例如有多少节点和边，feature和label的维度等。因为之前已经生成好graph了，所以获取这类信息只需要调用.number_of_nodes()等方法。

model初始化

然后初始化model的一些参数，例如设备，优化器，损失函数等等。这里比较常规就不展开说。

各类初始化工作完成之后就进入训练过程。

Data split and run

首先如果是选择tran模式，实际调用了run_transductive()。这个函数也是根据模型的不同有不同的操作‘。
首先确定indices有三种，test，train和val。
然后传递参数进入run_transductive()。

开头还是基本的初始化。接着根据model选择的不同分为三种情况：sage，mlp和其他。
例如mode name是gcn，那么直接调用train函数，返回loss。（Input the entire graph g as data.）
后面的criterion等也比较常规，接着就是每五轮评估一次，然后输出最好的acc。

ind模式其实就是在数据集中多加了一个obs g（就是利用已有的graph生成的新graph）

结果输出&保存
