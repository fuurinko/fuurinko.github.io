菜鸟云GPU方案

腾讯云

随便点了一个四十五块十五天的套餐，不过我知道十五天后不可能是这个价，虽然速度很垃圾但是便宜。

然后安装jupyter以及一些必须的裤。
jupyter还是很舒服，但是如果开了梯子就慢成树懒。
性能的话d2l里Alexnet跑mnish 也不快，但至少比colab无gpu或者可怜的mbp inter的cpu跑得快。
我下午还抱有一丝侥幸心理想说不定我电脑撑得住呢。结果快二十分钟一个epoch都没有跑完，cpu python占用率99.9%。。

omg，这个好像移动那种一块钱订阅，12月起订不得取消续费。。

这个方案基本作废，性价比奇低。

关于jupyter的配置问题，有两个解决思路。

ipykernel添加conda环境

这个比较常规。
基本方案是
1) 在默认bash中conda init
2) conda create+conda activate，常规 解法。
3) conda install ipykernel，ipython kernel install --user --name=d2l
4) 然后利用conda install pytorch,pip install d2l等安装库
5) 直接点开jupyter lab，选择启动内核为d2l。
目前这个方案是可行的，包什么的都能导入，但是就在我准备测试代码的时候，autodl崩了，速度血慢，遂放弃，明天再来。

暴力开启jupyter服务

这个方法是看别人博客上的，基本思路就是按照普通的方式起jupyter服务。本来这应该是基本操作，但是由于autodl并没有那种可以给你起服务的公网ip，并且开启那个所谓的自定义服务以后，给你的端口还是固定的，所以需要修改jupyter默认的端口。

总结

一套无坑配置：
拿到新的机子（以3090为例），ssh连接或者用平台提供的jupyter lab的终端也可以。

  1. 切到/minicoda/bin文件夹，conda init初始化
  2. 重启终端，按照d2l环境配置：conda create --name d2l python=3.9 -y，接着conda activate d2l。
  3. 然后这里，由于这个用户默认为root，如果嗯装什么包他不会给你报错，但是亲测装了等于没有装，于是再起一个python的虚拟环境，也就是python -m venv tutorial-env，source tutorial-env/bin/activate。
  4. 然后安装torch，这里是坑最多的，3090的话直接pip3 install torch==1.7.1+cu110 torchvision==0.8.2+cu110 -f https://download.pytorch.org/whl/cu110/torch_stable.html
  5. 然后pip install d2l。
  6. 这个时候基本上就可以在bash里跑py了，但是非常不优雅。于是下一步是配置jupyter。
  7. 大致步骤
